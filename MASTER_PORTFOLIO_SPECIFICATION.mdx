## Executive Vision

### The Product

This is not a portfolio website. It is a **personal brand product** — a cinematic, interactive experience that positions Sajjad Karim as a **Product Engineer** who ships premium digital experiences at the intersection of frontend craft, full-stack architecture, and product thinking.

### The Reaction We Optimize For

> *"This developer operates at an elite level."*

Recruiters, clients, and senior engineers should feel this within **3 seconds** of landing — before reading a single paragraph.

### What We Are Replacing

The current site is a customized open-source template with:
- Code-editor gimmick as dominant motif
- Broken mobile navigation
- Template footer attribution
- No motion system, no design tokens, no case studies
- Misconfigured contact pipeline

### What We Are Building

A **timeless, premium, cinematic** experience with:
- Typography-led hero with surgical motion
- Visual case-study project showcase
- Unified career timeline
- Glass-surface navigation with scroll intelligence
- Server-first architecture with client islands for motion only
- Lighthouse 100 targets across all categories

### Anti-Patterns (Explicitly Forbidden)

| Forbidden | Reason |
|-----------|--------|
| Generic Tailwind landing page aesthetic | Signals template |
| Infinite skills marquee | 2020 portfolio cliché |
| macOS terminal window as hero centerpiece | Overused; demote to micro-signature |
| Pink/violet gradient buttons on every CTA | Template tell |
| Inter as sole font | Default internet font since 2020 |
| Lottie animations in every section | Visual noise |
| Copying Linear/Vercel layouts pixel-for-pixel | Plagiarism, not inspiration |
| WebGL hero + particles + cursor trail + parallax everywhere | Gimmick overload |
| Light mode as afterthought | If included, must be fully tokenized |

### Unique Identity Anchor

**Codename: Aperture**

The design system is internally named **Aperture** — representing *focus*, *depth*, and *intentional framing*. Every section "frames" Sajjad's work like a camera aperture isolates subject from background. The experience feels **cinematic** but **restrained**: one hero moment, one scroll signature, one hover language — repeated with discipline.

## Design Philosophy

Every decision must answer: **"Does this increase perceived craft, or does it add noise?"**

### Principles

| Principle | Definition | Application |
|-----------|------------|-------------|
| **Minimal** | Remove until nothing meaningful can be removed | Max 7 homepage sections; no decorative Lottie |
| **Luxury** | Generous whitespace, precise typography, restrained color | 80px+ section gaps on desktop; single accent color |
| **Modern** | 2026 engineering standards: RSC, Motion, tokens | Server Components default; client only for interactivity |
| **Future-proof** | TypeScript, token-driven, versioned design system | No hardcoded hex in components |
| **Elegant** | Transitions feel inevitable, not flashy | 400ms ease-out; no bounce easing on UI |
| **High-end** | Details invisible until noticed | 1px hairline borders, film grain at 2% opacity |
| **Creative** | One signature moment users remember | Hero typographic reveal + project hover depth |
| **Interactive** | Response to input within 16ms perception | Hover states, magnetic CTAs, scroll reveals |
| **Fast** | Performance is a feature | LCP < 1.2s; no animation on critical path |
| **Accessible** | WCAG 2.1 AA minimum; AAA where feasible | Reduced motion, focus rings, semantic HTML |
| **Professional** | Conversion-optimized for hiring and clients | Clear CTAs, case studies, availability status |

### Decision Framework

When evaluating any design or engineering choice, score 1–5 on:

1. **Craft** — Does it demonstrate senior-level attention?
2. **Clarity** — Can a recruiter understand value in 6 seconds?
3. **Performance** — Does it cost more than 5KB or 1ms?
4. **Originality** — Would this appear on 100 other portfolios?
5. **Accessibility** — Does it exclude anyone?

**Minimum acceptable total: 18/25.** Reject anything below.

### Signature Design Language

**The Aperture Language — 5 Signatures:**

1. **Obsidian Canvas** — Near-black backgrounds with subtle blue undertone (not purple template)
2. **Mint Signal** — Single accent `#3DFFB8` (evolved from current `#16f2b3`, slightly desaturated for luxury)
3. **Hairline Frame** — 1px borders at `rgba(255,255,255,0.08)` on all elevated surfaces
4. **Display + Mono Pairing** — Serif/sans display for headlines; monospace for metadata
5. **Depth on Intent** — Scale, blur, and shadow activate only on hover/focus — never at rest

**Micro-signature (optional, max 1 per page):**

A single-line monospace prompt in footer or hero corner:

```
sajjad@portfolio ~ ready
```

No blinking cursor animation. Static or subtle opacity pulse only.

### Brand Consistency Rules

1. Accent color appears **maximum 3 times** per viewport (logo dot, one CTA, one data highlight)
2. Gradients limited to **hero background mesh** and **one CTA variant** — nowhere else
3. All caps only for **section eyebrows** (11px, tracking 0.2em) — never headlines
4. Photography: desaturated 10%, consistent border-radius `radius-image`
5. Third-party logos (skills, companies): monochrome at 60% opacity, color on hover only

### Unique Identity System

**Logo mark:** Wordmark "Sajjad Karim" in display font. Preceded by a 6px mint square (brand dot) — not a `{ }` or `< />` icon.

**Favicon:** Mint square on obsidian background.

**OG Image:** Display headline + brand dot + subtle grid. No photo (photo ages; typography endures).

### Typography

#### Font Pairing (Recommended)

| Role | Font | Source | Fallback |
|------|------|--------|----------|
| **Display** | **Instrument Serif** | Google Fonts / next/font | Georgia, serif |
| **Body** | **Geist Sans** | Vercel CDN / next/font | system-ui, sans-serif |
| **Mono** | **Geist Mono** | Vercel CDN / next/font | ui-monospace, monospace |

**Why this pairing:**
- Instrument Serif: editorial luxury, distinct from Inter/Geist-only portfolios
- Geist Sans: technical credibility, Vercel-adjacent without copying their layout
- Geist Mono: metadata, tags, terminal micro-signature

**Alternative if Instrument Serif unavailable:** **Fraunces** (display) + Geist Sans (body)

When `prefers-reduced-motion: reduce`:
- All durations → `--duration-instant`
- Scroll reveals → opacity only, no translateY
- Hero stagger → simultaneous fade-in
- Lenis smooth scroll → disabled
- Parallax → disabled
- Magnetic effects → disabled

## Motion System

### Philosophy

Motion communicates **hierarchy and causality** — not decoration. Every animation answers: "What changed, and why should the user care?"

**Budget:** Maximum 60fps on mid-range Android. Total client JS for motion < 45KB gzip.

### Page Transitions

| Transition | Spec |
|------------|------|
| Route change | Fade out current (150ms) → fade in next (250ms) via `AnimatePresence mode="wait"` |
| Shared element | Project card image → case study hero via `layoutId` (optional Phase 4) |
| Scroll position | Reset to top on route change via Lenis |

### Hero Timeline (0–900ms)

| Time | Element | Animation |
|------|---------|-----------|
| 0ms | Background mesh | Opacity 0→1, 600ms |
| 100ms | Eyebrow text | Opacity 0→1, translateY 12→0, 400ms |
| 200ms | Headline word 1 | Stagger per word, 80ms delay each |
| 400ms | Subheadline | Opacity 0→1, translateY 8→0, 400ms |
| 500ms | CTA group | Opacity 0→1, translateY 8→0, 400ms |
| 600ms | Stats strip | Opacity 0→1, 400ms |
| 700ms | Scroll indicator | Opacity 0→0.6, subtle translateY loop (2s repeat) |

**Skip if reduced motion:** All elements fade in simultaneously at 200ms.

### Mouse Movement & Cursor

| Feature | Spec | Scope |
|---------|------|-------|
| Custom cursor | **No** — system cursor preserved for accessibility | — |
| Magnetic button | Primary CTA follows cursor within 8px radius | Desktop only, `@media (hover: hover)` |
| Spotlight | **No** — replaced by card hover glow | — |
| Parallax hero mesh | Background gradient shifts ±2% based on mouse position | Desktop only, subtle |

### Card Hover

```
Rest → Hover (400ms ease-out):
- translateY: 0 → -4px
- border-color: border-default → border-strong
- shadow: elevation-1 → elevation-3
- Optional: inner image scale 1 → 1.03 (overflow hidden)
```

Project cards additionally reveal:
- Overlay gradient from bottom (opacity 0→0.6)
- "View case study →" text slides up 8px

### Button Hover

| Variant | Hover |
|---------|-------|
| Primary | Background lightens 5%; shadow-glow-primary; scale 1.02 |
| Secondary | Border → border-accent; background surface-hover |
| Ghost | Text → primary; underline slides in from left |
| Icon | Background surface-hover; icon color → primary |

Press: scale 0.98, 150ms

### Image Hover

- Project thumbnails: scale 1.03 inside overflow:hidden container, 600ms ease-out
- Profile photo: no scale — static with subtle border glow on section hover only

### Scroll Reveal

Default section reveal via `ScrollReveal` wrapper:

```typescript
// Spec (not implementation):
initial: { opacity: 0, y: 24 }
whileInView: { opacity: 1, y: 0 }
viewport: { once: true, margin: "-10%" }
transition: { duration: 0.6, ease: ease-out }
```

Stagger children within section: 80ms delay per child.

### Parallax

| Element | Rate | Phase |
|---------|------|-------|
| Hero mesh background | 0.3x scroll speed | Phase 3 |
| Section decorative elements | **None** | — |

Parallax disabled on mobile and reduced-motion.

### Depth & Layering

| Z-index | Layer |
|---------|-------|
| 0 | Canvas background, mesh |
| 10 | Content |
| 20 | Sticky cards |
| 50 | Navbar |
| 60 | Mobile drawer |
| 70 | Toast |
| 80 | Modal/dialog |
| 90 | Skip link (focus) |

### Physics

- Spring physics for: mobile drawer, modal enter, toast
- Tween physics for: hover, scroll reveal, hero stagger
- No bounce easing on UI elements (bounce reads as playful, not premium)

### Background Animation

| Effect | Spec |
|--------|------|
| Hero mesh | CSS radial gradients, optional slow 20s hue rotate on secondary orb |
| Film grain | Fixed `noise.svg` overlay, 2% opacity, pointer-events: none |
| Particles | **Not approved for v1** — revisit only if hero feels empty in user testing |
| Aurora | **Not approved** — too trendy, dates quickly |

### Glass Morphism

| Element | Spec |
|---------|------|
| Navbar (scrolled) | glass-bg + glass-blur + glass-border + shadow-glass |
| Navbar (top) | Transparent, no blur |
| Mobile drawer | glass-bg at 95% opacity |
| Cards | **No glass** — solid surfaces only |

```css
/* Spec */
focus-visible: {
  outline: 2px solid var(--color-primary);
  outline-offset: 2px;
}
```

Never remove focus outlines. Custom focus ring matches brand.

### Contrast

| Pairing | Ratio Target |
|---------|-------------|
| Body text on canvas | ≥ 7:1 (AAA) |
| Muted text on canvas | ≥ 4.5:1 (AA) |
| Primary button text | ≥ 4.5:1 |

Validate all token pairings in Phase 2.

Global CSS:
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;

  }
}
```

Plus: disable Lenis, magnetic, parallax via `useReducedMotion` hook.

- H1 once per page
- Heading hierarchy never skips levels
- Images: descriptive alt text
- Decorative images: `alt=""`
- Form errors announced via live region
